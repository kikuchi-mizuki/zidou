# 海外高級チョコレート納品計画システム 使い方ガイド

## 🍫 取扱商品について

本システムは、以下の**海外輸入高級チョコレート（専門店向けプレミアムブランド）**の卸業務に対応しています：

### 取扱ブランド
- **Amedei（アメデイ）** - イタリア・トスカーナ / Bean to Bar / 単一農園カカオ
- **Domori（ドモーリ）** - イタリア・ピエモンテ / クリオーロ種100%
- **Michel Cluizel（ミシェル・クルイゼル）** - フランス / 自社農園カカオ
- **Bonnat（ボナ）** - フランス・ヴォワロン / シングルオリジン
- **Coppeneur（コペヌール）** - ドイツ / 手作業製造
- **Friis-Holm（フリス・ホルム）** - デンマーク / 少量生産

### 輸入品管理の特徴
- **ロット管理**: 国別・ブランド別の厳格な管理
  - 例: ITA202601A = イタリア・2026年1月輸入・Amedei
- **輸送方法**: 航空便（賞味期限重視）/海上便（コスト重視）
- **少量生産品**: 入荷数が限定されるため、在庫管理が重要
- **高付加価値**: 百貨店ギフトコーナー・高級食品専門店向け
- **季節需要**: バレンタイン、ホワイトデー、母の日、クリスマス等

## 基本的な業務フロー

```
① 発注受付 → ② 在庫確認 → ③ 自動計算 → ④ 計画確認 → ⑤ 承認・出力
```

### ① 発注受付（発注管理）
デパートからの発注を登録

### ② 在庫確認（在庫管理）
在庫が足りるか確認、不足なら追加

### ③ 自動計算（自動計算）
納品計画を自動生成

### ④ 計画確認（納品計画）
生成された計画を確認

### ⑤ 承認・出力（納品計画）
計画を承認してExcel出力

## 各画面の使い方

### 1. ダッシュボード (`/`)

**機能**: システム全体の状況を一目で確認

**表示内容**:
- 発注件数: 処理待ちの発注数
- 在庫アラート: 10ケース未満の商品数
- 期限が近い在庫: 60日以内に期限切れとなる在庫
- 商品数: 登録商品の総数

**使い方**:
1. 各カードで状況を確認
2. 「期限が近い在庫」で優先処理が必要な在庫を把握
3. 「処理待ち発注」で対応が必要な発注を確認
4. 「自動計算を実行」ボタンで計算画面へ

---

### 2. 商品管理 (`/products`)

**機能**: 取り扱い商品の一覧表示

**表示項目**:
- 商品コード（例: CHO-001）
- 商品名
- ブランド名
- ケース入数

**現在のデモ版の制限**:
- 商品の追加・編集・削除は未実装
- 初期データのみ表示

---

### 3. 在庫管理 (`/inventory`)

**機能**: 在庫の確認・追加・編集

#### 商品別在庫サマリー
- 各商品の総在庫ケース数を表示
- 10ケース未満は「在庫少」バッジで警告

#### 在庫一覧テーブル
**表示項目**:
- 商品（商品名 + コード）
- ロット番号
- 賞味期限
- ケース数（直接編集可能）
- 状態（残日数による色分け）
  - 🔴 赤: 30日以内
  - 🟡 黄: 31-60日
  - 🟢 緑: 61日以上

#### 在庫追加の手順
1. 「在庫追加」ボタンをクリック
2. 以下を入力:
   - 商品: プルダウンから選択
   - ロット番号: 例「LOT5」
   - 賞味期限: 日付選択
   - ケース数: 数値入力
3. 「登録」ボタンで追加

#### ケース数の編集
- テーブル内の数値入力欄で直接編集
- 変更は即座に反映（localStorageに保存）

#### 在庫の削除
- 「削除」ボタンをクリック
- 確認ダイアログで「OK」

---

### 4. 発注管理 (`/orders`)

**機能**: デパートからの発注を管理

#### 発注一覧テーブル
**表示項目**:
- 発注日
- デパート名
- 商品
- 数量（個数 + 必要ケース数）
- 希望納品日
- ステータス
  - 🟡 処理待ち: pending
  - 🔵 処理中: processing
  - 🟢 完了: completed
- 操作（削除ボタン）

#### 新規発注の登録手順
1. 「新規発注」ボタンをクリック
2. 以下を入力:
   - デパート名: プルダウンから選択
     - 高島屋、伊勢丹、三越、大丸、松坂屋
   - 商品: プルダウンから選択
   - 数量（個数）: 例「100」
   - 希望納品日: 日付選択
3. 「登録」ボタンで追加
4. ステータスは自動的に「処理待ち」に

#### 発注の削除
- 「処理待ち」状態の発注のみ削除可能
- 「削除」ボタン → 確認ダイアログで「OK」

---

### 5. ルール管理 (`/rules`)

**機能**: 納品計画の自動計算ルールを管理

#### 登録済みルール
デフォルトで以下のルールが設定済み:

1. **賞味期限優先**
   - 賞味期限が近い在庫を優先的に使用する

2. **納品時30日以上**
   - 納品時に賞味期限が30日以上残っている必要がある

3. **高島屋納品曜日**
   - 高島屋は火曜日と金曜日のみ納品可能

4. **1回最大20ケース**
   - 1回の納品で最大20ケースまで

#### ルールの有効/無効切り替え
- スイッチをクリックで即座に切り替え
- 無効化されたルールは自動計算時に適用されない

#### 新しいルールの追加手順
1. 「ルール追加」ボタンをクリック
2. 以下を入力:
   - ルール名: 例「伊勢丹納品曜日」
   - 説明: 例「伊勢丹は月曜日と木曜日のみ納品可能」
3. 「追加」ボタンで登録
4. 新しいルールはデフォルトで有効

**注意**:
- デモ版では、追加したルールは画面表示のみ
- 実際の計算ロジックには反映されません
- 本番版では、ルールエンジンと連携して動作します

### 6. 自動計算 (`/calculate`)

**機能**: 発注と在庫を基に最適な納品計画を自動生成

#### 計算実行前の確認
**処理対象の表示**:
- 処理待ち発注: X件
- 在庫ロット数: X件
- 有効なルール: X件

#### 自動計算の実行手順
1. 処理対象を確認
2. 「自動計算を実行」ボタンをクリック
3. 計算ログをリアルタイムで確認
4. 完了メッセージが表示されたら「納品計画を確認」ボタンをクリック

#### 計算ログの見方
各ログには以下が表示されます:
- アイコン（情報/警告/成功/エラー）
- ステップ名（バッジ表示）
- タイムスタンプ
- メッセージ内容

**ログタイプ**:
- 🔵 情報: 処理の進行状況
- 🟡 警告: 注意が必要な状況
- 🟢 成功: 処理完了
- 🔴 エラー: 問題発生

#### 計算処理の内容
1. **初期化**: 処理対象の発注件数を確認
2. **在庫確認**: 各商品のロット数と総ケース数を確認
3. **在庫不足チェック**: 必要ケース数と在庫を比較
4. **納品日計算**: デパート別の納品可能曜日を考慮
5. **期限チェック**: 賞味期限30日以上の制約を確認
6. **計画作成**: ロットを割り当てて計画を生成
7. **完了**: 生成された計画件数を報告

#### エラーが発生した場合
**在庫不足エラー**:
```
エラー: ダークチョコレート70%: 必要10ケース、在庫5ケース（不足5ケース）
```
→ 在庫管理画面で在庫を追加してください

**賞味期限エラー**:
```
警告: ロットLOT1: 納品時残り25日（30日未満のためスキップ）
```
→ 該当ロットは使用されません。新しい在庫を追加するか、希望納品日を早めてください

---

### 7. 納品計画 (`/delivery-plan`)

**機能**: 自動生成された納品計画の確認・承認・出力

#### 納品計画サマリー
- 総納品回数
- 納品先の数
- 総ケース数

#### 納品日別の計画表示
**表示形式**:
```
2026年4月4日（金曜日）
┌──────────────────────────────────┐
│ デパート │ 商品 │ ケース数 │ ロット │ 賞味期限 │ 状態 │
├──────────────────────────────────┤
│ 高島屋   │ ダーク │ 5ケース │ LOT1  │ 2026-04-30 │ ⚠️  │
└──────────────────────────────────┘
```

#### 状態バッジの意味
- 🟡 警告: 賞味期限まで45日以内
- 🟢 正常: 賞味期限まで46日以上

#### 計画の承認手順
1. 計画内容を確認
2. 問題がなければ「計画を承認」ボタンをクリック
3. 確認ダイアログで「OK」
4. 承認完了メッセージが表示される

#### Excel出力（デモ）
1. 「Excel出力」ボタンをクリック
2. デモメッセージが表示される

**本番版では**:
- 実際のExcelファイルが生成される
- 含まれる情報:
  - 納品日
  - デパート名・住所
  - 商品詳細
  - ケース数・個数
  - ロット番号
  - 賞味期限
  - 配送指示事項

## 実際の業務シナリオ例

### シナリオ1: 通常の発注処理（イタリア産プレミアムチョコレート）

#### 状況
高島屋の母の日ギフトコーナーから「アメデイ トスカーノブラック70% 100個」の発注が入った。
希望納品日は2026年4月8日。

#### 背景
- **商品**: Amedei（イタリア・トスカーナの老舗）のBean to Barチョコレート
- **特徴**: 単一農園カカオを使用した高級品
- **価格帯**: 1個あたり800-1,200円（小売価格）
- **用途**: 母の日ギフトセット用

#### 手順

1. **発注登録**（発注管理画面）
   - 「新規発注」をクリック
   - デパート: 高島屋
   - 商品: CHO-001 - アメデイ トスカーノブラック70%
   - 数量: 100
   - 希望納品日: 2026-04-08
   - 「登録」をクリック

2. **在庫確認**（在庫管理画面）
   - アメデイ トスカーノブラックの在庫を確認
   - 必要ケース数: 100個 ÷ 12個/ケース = 9ケース（切り上げ）
   - 在庫:
     - ITA202601A（2026年1月航空便輸入）5ケース
     - ITA202602A（2026年2月航空便輸入）8ケース
     - 合計: 13ケース
   - ✅ 在庫は十分

3. **自動計算実行**（自動計算画面）
   - 「自動計算を実行」をクリック
   - ログを確認:
     ```
     ✓ 在庫確認: アメデイ トスカーノブラック70%: 2ロット、合計13ケース
     ✓ 納品日計算: 高島屋の納品可能日: 2026-04-04, 2026-04-08
     ⚠ 期限チェック: ロットITA202601A（イタリア2026年1月輸入）: 納品時残り26日
     ✓ 計画作成: 2026-04-04: アメデイ トスカーノブラック70% 5ケース（ロットITA202601A）
     ✓ 計画作成: 2026-04-08: アメデイ トスカーノブラック70% 4ケース（ロットITA202602A）
     ✓ 完了: 納品計画を2件作成しました
     ```

4. **計画確認**（納品計画画面）
   - 2回に分けて納品する計画を確認
   - 1回目（4/4金曜）: ITA202601A（1月輸入ロット）を使い切る
     - 理由: 賞味期限が近いため優先消化
     - 警告: 納品時残り26日（ギリギリ30日ルールをクリア）
   - 2回目（4/8火曜）: ITA202602A（2月輸入ロット）から残り4ケース
     - 賞味期限に余裕あり

5. **承認・出力**
   - 計画を承認
   - Excel出力で配送指示書を作成

### シナリオ2: 在庫不足の場合

#### 状況
伊勢丹から「ミルクチョコレート 150個」の発注。
希望納品日は2026年4月10日。

#### 手順

1. **発注登録**
   - デパート: 伊勢丹
   - 商品: CHO-002 - ミルクチョコレート
   - 数量: 150
   - 希望納品日: 2026-04-10

2. **在庫確認**
   - 必要ケース数: 150個 ÷ 10個/ケース = 15ケース
   - 現在の在庫: LOT3 12ケース
   - ❌ 在庫不足（3ケース不足）

3. **在庫追加**（在庫管理画面）
   - 「在庫追加」をクリック
   - 商品: CHO-002
   - ロット番号: LOT5
   - 賞味期限: 2026-07-31
   - ケース数: 5
   - 「登録」をクリック

4. **再度在庫確認**
   - 総在庫: 12 + 5 = 17ケース
   - ✅ 在庫十分

5. **自動計算実行**
   - ログで在庫が確保されたことを確認
   - 計画が正常に作成される

---

### シナリオ3: 大量発注の分割納品

#### 状況
三越から「オレンジチョコレート 200個」の大量発注。
希望納品日は2026年4月12日。

#### 手順

1. **発注登録**
   - デパート: 三越
   - 商品: CHO-003 - オレンジチョコレート
   - 数量: 200
   - 希望納品日: 2026-04-12

2. **必要ケース数の計算**
   - 必要: 200個 ÷ 8個/ケース = 25ケース
   - ルール「1回最大20ケース」により、2回以上に分割必要

3. **自動計算の結果**
   ```
   計画1: 2026-04-10（水）20ケース
   計画2: 2026-04-13（土）5ケース
   ```
   - 三越の納品可能曜日: 水曜・土曜
   - 最大20ケースのルールで自動分割

4. **計画確認**
   - 2回の納品計画を確認
   - 各回の詳細を確認
   - 問題なければ承認

## トラブルシューティング

### Q1: 発注や在庫を追加したのに、自動計算に反映されない
**原因**: ブラウザのlocalStorageに保存されているデータと画面の表示が同期していない

**解決方法**:
1. ページをリロード（F5キー）
2. それでも反映されない場合は、ブラウザの開発者ツールを開く
3. Application > Local Storage > localhost:3000 を確認
4. `orders`、`inventory` のデータを確認

---

### Q2: 計算ログで「在庫不足」エラーが出る
**原因**: 発注数量に対して在庫が足りない

**解決方法**:
1. エラーログで不足ケース数を確認
2. 在庫管理画面で該当商品の在庫を追加
3. 自動計算を再実行

---

### Q3: 計算ログで「賞味期限」警告が出る
**原因**: すべての在庫ロットが「納品時30日以上」の条件を満たさない

**解決方法**:
1. 新しい在庫（賞味期限が長いもの）を追加
2. または希望納品日を早める
3. ルール管理で「納品時30日以上」を一時的に無効化（非推奨）

---

### Q4: 納品計画が表示されない
**原因**: 自動計算がまだ実行されていない

**解決方法**:
1. 発注管理で発注を登録
2. 自動計算画面で計算を実行
3. 完了後に納品計画画面へ

---

### Q5: データが消えてしまった
**原因**: ブラウザのlocalStorageがクリアされた

**解決方法**:
1. ページをリロードすると初期データが表示される
2. 追加したデータは復元できないため、再度登録が必要
3. 本番版ではデータベースに保存されるため、この問題は発生しません

---

### Q6: デパート別の納品曜日ルールを変更したい
**現在の設定**:
- 高島屋: 火曜・金曜
- 伊勢丹: 月曜・木曜
- 三越: 水曜・土曜
- 大丸: 月水金
- 松坂屋: 火曜・木曜

**デモ版での変更方法**:
1. `lib/calculation.ts` を編集
2. `DELIVERY_RULES` オブジェクトを変更
   ```typescript
   const DELIVERY_RULES: Record<string, number[]> = {
     "高島屋": [2, 5], // 火曜・金曜
     // 変更例: 月曜・水曜に変更
     "高島屋": [1, 3],
   };
   ```
3. サーバーを再起動

**本番版では**:
- ルール管理画面から設定可能

## システムの制限事項（デモ版）

### データの永続化
- localStorageを使用しているため、ブラウザのキャッシュをクリアするとデータが消える
- 本番版ではデータベースに保存

### 同時ユーザー
- 複数ユーザーでの同時利用には非対応
- 本番版では認証・権限管理を実装

### Excel出力
- ダミー実装（アラート表示のみ）
- 本番版では実際のExcelファイルを生成

### ルールエンジン
- 追加したルールは表示のみ
- 計算ロジックには標準ルールのみ適用
- 本番版では動的にルールを適用
