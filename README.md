# チョコレート卸向け納品計画WEBアプリ

## システム概要

### 目的
チョコレート卸売業者が、デパートからの発注に対して効率的な納品計画を作成するためのツールです。

### 解決する課題
- ✅ 賞味期限が近い在庫の優先消化
- ✅ デパート別の納品曜日ルールの遵守
- ✅ 手作業による計画作成の時間削減
- ✅ 在庫状況の可視化

### 業務フロー
```
発注受付 → 在庫確認 → 自動計算 → 計画確認 → 承認・出力
```

---

## 主な機能

### 1. ダッシュボード
- 発注件数、在庫アラート、期限が近い在庫の一覧表示
- システム全体の状況を一目で把握

### 2. 商品管理
- 取り扱い商品の一覧表示
- 商品コード、商品名、ブランド、ケース入数

### 3. 在庫管理
- ✨ **在庫の追加・編集・削除**
- ✨ **ケース数の直接編集**
- 商品別在庫サマリー表示
- 賞味期限による警告表示（30日/60日）
- 在庫不足アラート

### 4. 発注管理
- ✨ **新規発注の登録**
- ✨ **発注の削除**
- 発注ステータス管理（処理待ち/処理中/完了）
- 必要ケース数の自動計算

### 5. ルール管理
- ✨ **ルールの追加**
- ✨ **ルールの有効/無効切り替え**
- 標準ルール:
  - 賞味期限優先
  - 納品時30日以上
  - デパート別納品曜日
  - 1回最大20ケース

### 6. 自動計算
- ✨ **実際の計算ロジック実装**
- リアルタイム計算ログ表示
- エラーハンドリング（在庫不足、賞味期限）
- 発注ステータスの自動更新

### 7. 納品計画
- ✨ **日付別グループ表示**
- ✨ **計画の承認機能**
- 納品計画サマリー
- Excel出力（デモ）

## 使い方

### クイックスタート

#### 1. 発注を登録する
1. 左メニューから「発注管理」を開く
2. 「新規発注」ボタンをクリック
3. デパート名、商品、数量、希望納品日を入力
4. 「登録」をクリック

#### 2. 在庫を確認する
1. 左メニューから「在庫管理」を開く
2. 商品別の在庫サマリーを確認
3. 不足している場合は「在庫追加」で追加

#### 3. 納品計画を作成する
1. 左メニューから「自動計算」を開く
2. 処理対象を確認
3. 「自動計算を実行」をクリック
4. 計算ログを確認

#### 4. 計画を確認・承認する
1. 左メニューから「納品計画」を開く
2. 日付別の計画を確認
3. 「計画を承認」をクリック
4. 「Excel出力」で出力（デモ）

### 詳細な使い方
詳しい使い方は [使い方ガイド.md](./使い方ガイド.md) を参照してください。

## デモ版の特徴

### データの永続化
- **localStorageを使用**
  - ブラウザを閉じてもデータは保持される
  - ブラウザのキャッシュをクリアすると初期化される

### 実際の業務フロー
1. 発注を追加 → localStorageに保存
2. 在庫を追加 → localStorageに保存
3. 自動計算実行 → 実際のロジックで計画生成
4. 発注ステータス自動更新 → pending → processing
5. 計画承認 → 承認済みステータスに更新

### 計算ロジック
以下のルールを実際に適用:
- ✅ 賞味期限優先（古いロットから使用）
- ✅ 納品時30日以上の期限チェック
- ✅ デパート別納品曜日（高島屋: 火金、伊勢丹: 月木など）
- ✅ 1回最大20ケースの制限
- ✅ 在庫不足のエラー検出
- ✅ 賞味期限不足の警告
